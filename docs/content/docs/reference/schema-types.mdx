---
title: Schema column types
description: Reference for schema-mode column helpers and their select, insert, and predicate types.
---

Schema-mode column helpers are exported from the package root and produce
`QuarryColumn<Select, Insert, Where>` values.

- `Select` is the runtime TypeScript type you get back from `select(...)`
- `Insert` is the TypeScript type accepted in `insertInto(...).values(...)`
- `Where` is the TypeScript type accepted by predicates such as
  `.where("col", "=", value)`

A typical schema mixes scalar helpers such as `UInt64()` and `DateTime64(3)`
with wrappers such as `Nullable(...)`, `LowCardinality(...)`, and `Array(...)`.

<Callout type="info">
Quarry does not yet export `Decimal*` schema helpers. You can still cast query
expressions with `eb.fn.toDecimal64(...)` and `eb.fn.toDecimal128(...)`; see
[Schema mode and views](/docs/guides/schema-mode-and-views#current-limits).
</Callout>

## Scalar helpers

| Helper | ClickHouse type | Select type | Insert type | Predicate type | Example |
| --- | --- | --- | --- | --- | --- |
| `String()` | `String` | `string` | `string` | `string` | `email: String()` |
| `Bool()` | `Bool` | `boolean` | `boolean` | `boolean` | `is_active: Bool()` |
| `UInt8()` | `UInt8` | `number` | `number` | `number` | `shard: UInt8()` |
| `UInt16()` | `UInt16` | `number` | `number` | `number` | `bucket: UInt16()` |
| `UInt32()` | `UInt32` | `number` | `number` | `number` | `user_id: UInt32()` |
| `UInt64()` | `UInt64` | `string` | `string \| number \| bigint` | `string \| number \| bigint` | `event_id: UInt64()` |
| `Int8()` | `Int8` | `number` | `number` | `number` | `delta: Int8()` |
| `Int16()` | `Int16` | `number` | `number` | `number` | `offset: Int16()` |
| `Int32()` | `Int32` | `number` | `number` | `number` | `score: Int32()` |
| `Int64()` | `Int64` | `string` | `string \| number \| bigint` | `string \| number \| bigint` | `balance: Int64()` |
| `Float32()` | `Float32` | `number` | `number` | `number` | `ratio: Float32()` |
| `Float64()` | `Float64` | `number` | `number` | `number` | `amount: Float64()` |
| `FixedString(n)` | `FixedString(n)` | `string` | `string` | `string` | `country_code: FixedString(2)` |
| `UUID()` | `UUID` | `string` | `string` | `string` | `request_id: UUID()` |
| `IPv4()` | `IPv4` | `string` | `string` | `string` | `client_ip: IPv4()` |
| `IPv6()` | `IPv6` | `string` | `string` | `string` | `client_ip_v6: IPv6()` |

## Date and time helpers

| Helper | ClickHouse type | Select type | Insert type | Predicate type | Example |
| --- | --- | --- | --- | --- | --- |
| `Date()` | `Date` | `string` | `string \| Date` | `string \| Date` | `signup_date: Date()` |
| `Date32()` | `Date32` | `string` | `string \| Date` | `string \| Date` | `cohort_day: Date32()` |
| `DateTime()` | `DateTime` | `string` | `string \| Date` | `string \| Date` | `created_at: DateTime()` |
| `DateTime64(p)` | `DateTime64(p)` | `string` | `string \| Date` | `string \| Date` | `created_at: DateTime64(3)` |

When you pass a JavaScript `Date` to a schema-backed insert or predicate,
Quarry formats it according to the column type before sending it to
ClickHouse.

## Wrapper helpers

| Helper | ClickHouse type | Select type | Insert type | Predicate type | Example |
| --- | --- | --- | --- | --- | --- |
| `Nullable(T)` | `Nullable(inner)` | `InnerSelect \| null` | `InnerInsert \| null` | `InnerWhere \| null` | `deleted_at: Nullable(DateTime64(3))` |
| `LowCardinality(T)` | `LowCardinality(inner)` | `InnerSelect` | `InnerInsert` | `InnerWhere` | `event_type: LowCardinality(String())` |
| `Array(T)` | `Array(inner)` | `InnerSelect[]` | `InnerInsert[]` | `InnerWhere[]` | `tags: Array(String())` |

`LowCardinality(...)` changes the ClickHouse type but preserves the inner
TypeScript types. `Nullable(...)` adds `null`. `Array(...)` lifts the inner
select, insert, and predicate types into arrays.
