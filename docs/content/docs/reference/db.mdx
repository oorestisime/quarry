title: createClickHouseDB
description: The entry point that turns a @clickhouse/client instance into a typed database handle.
---

`createClickHouseDB(...)` is the entry point of the library. It supports two
ways of describing your schema:

- a plain TypeScript interface
- a richer schema object built with `defineSchema(...)`

Both return the same `db` handle shape.

## Plain mode

```ts
import { createClient } from "@clickhouse/client";
import { createClickHouseDB } from "@oorestisime/quarry";

interface DB {
  users: {
    id: number;
    email: string;
  };
}

const db = createClickHouseDB<DB>({
  client: createClient({ url: "http://localhost:8123" }),
});
```

## Schema mode

```ts
import { createClient } from "@clickhouse/client";
import {
  DateTime64,
  String,
  UInt32,
  createClickHouseDB,
  defineSchema,
  table,
  view,
} from "@oorestisime/quarry";

const schema = defineSchema({
  users: table.replacingMergeTree({
    id: UInt32(),
    email: String(),
    created_at: DateTime64(3),
  }),
}).views((db) => ({
  active_users: view.as(
    db
      .selectFrom("users as u")
      .select("u.id", "u.email", "u.created_at")
  ),
}));

const db = createClickHouseDB({
  client: createClient({ url: "http://localhost:8123" }),
  schema,
});
```

## What you get back

`db` is an instance of `ClickHouseDB<...>`. The methods you actually use are:

- `db.selectFrom(...)` &mdash; start a `SELECT` query
- `db.insertInto(...)` &mdash; start an `INSERT` query
- `db.with(name, callback)` &mdash; add a CTE that the rest of the chain can
  reference
- `db.table(name)` &mdash; build an explicit table source (for `FINAL`,
  custom aliases, etc.)

Schema mode also lets you define query-backed views ahead of time with staged
schema construction:

```ts
const schema = defineSchema({
  users: table.replacingMergeTree({
    id: UInt32(),
    email: String(),
  }),
}).views((db) => ({
  user_emails: view.as(db.selectFrom("users as u").select("u.id", "u.email")),
}));
```

See the [guides](/docs/guides) for end-to-end examples of each.
